# PRD: Воркшоп "Правила, команди та навички для Agentic IDE"

## 1. Огляд проєкту

### 1.1 Мета

Створити комплексний воркшоп для досвідчених розробників про ефективну роботу з AI-асистентами в IDE (Agentic IDE), включаючи презентаційні матеріали, практичні вправи та додаткові ресурси.

### 1.2 Цільова аудиторія

- **Основна аудиторія:** Досвідчені розробники (senior level)
- **Передумови:** Досвід роботи з IDE, знання основ програмування
- **Очікувані знання:** Розуміння процесу розробки, робота з кодом

### 1.3 Тривалість та формат

- **Тривалість:** 1 година
- **Формат:** Комбінований (теорія 40% + практика 60%)
- **Структура:**
  - Вступ та теорія: ~25 хвилин
  - Інтерактивна демонстрація: ~30 хвилин
  - Q&A та висновки: ~5 хвилин

## 2. Вимоги до контенту

### 2.1 Презентаційні слайди (slides.md)

#### 2.1.1 Структура слайдів

1. **Титульний слайд**

   - Назва воркшопу
   - Інформація про ведучого
   - Дата та подія (FW Days 2026)

2. **Вступ (2-3 слайди)**

   - Що таке Agentic IDE?
   - Переваги використання
   - Статистика та тренди

3. **Правила роботи (3-4 слайди)**

   - Чіткість інструкцій
   - Ітеративний підхід
   - Контроль якості
   - Best practices

4. **Команди та інструкції (4-5 слайдів)**

   - Створення коду
   - Пошук та аналіз
   - Рефакторинг
   - Виправлення помилок
   - Приклади команд

5. **Навички ефективної роботи (3-4 слайди)**

   - Структурування запитів
   - Використання контексту
   - Редагування та уточнення
   - Практичні приклади

6. **Практична частина (2-3 слайди)**

   - Інтерактивна демонстрація
   - Живі приклади
   - Розв'язання реальних задач

7. **Висновки та ресурси (2 слайди)**

   - Ключові моменти
   - Додаткові ресурси
   - Q&A

#### 2.1.2 Технічні вимоги до слайдів

- Використання теми `seriph` або `default`
- Підтримка інтерактивних елементів (v-click, анімації)
- Вбудовані приклади коду з підсвіткою синтаксису
- Діаграми та візуалізації (Mermaid)
- Адаптивний дизайн для різних екранів

### 2.2 Практичні вправи

#### 2.2.1 Інтерактивна демонстрація

- **Формат:** Жива демонстрація роботи з AI-асистентом
- **Сценарії:**

  1. Створення React компонента з TypeScript
  2. Рефакторинг існуючого коду
  3. Виправлення помилок та покращення коду
  4. Оптимізація продуктивності

#### 2.2.2 Вправи для учасників

- Набір практичних завдань для самостійного виконання
- Приклади правильних та неправильних запитів
- Чеклист для самооцінки

### 2.3 Шпаргалка (Cheat Sheet)

#### 2.3.1 Структура шпаргалки

- **Розділ 1: Основні команди**
  - Створення коду
  - Рефакторинг
  - Пошук та аналіз
  - Виправлення помилок

- **Розділ 2: Патерни запитів**
  - Шаблони для різних типів задач
  - Приклади ефективних запитів
  - Типові помилки та як їх уникати

- **Розділ 3: Best Practices**
  - Правила формулювання запитів
  - Робота з контекстом
  - Контроль якості

#### 2.3.2 Формат

- PDF файл для друку
- Markdown версія для онлайн перегляду
- Компактний розмір (1-2 сторінки)

### 2.4 Додаткові ресурси

#### 2.4.1 Посилання та матеріали

- Офіційна документація AI-асистентів
- Статті та блоги
- Відео-туторіали
- GitHub репозиторії з прикладами

#### 2.4.2 Структура файлу ресурсів

- Категорії ресурсів
- Короткий опис кожного ресурсу
- Посилання та QR-коди

## 3. Технічні вимоги

### 3.1 Технологічний стек

- **Slidev** для презентації
- **Markdown** для контенту
- **Vue.js** для інтерактивних компонентів
- **Mermaid** для діаграм

### 3.2 Структура проєкту

```
/
├── slides.md              # Основна презентація
├── exercises/             # Практичні вправи
│   ├── exercises.md       # Опис вправ
│   └── solutions/         # Рішення (опціонально)
├── resources/             # Додаткові ресурси
│   ├── cheat-sheet.md     # Шпаргалка
│   ├── resources.md       # Список ресурсів
│   └── examples/          # Приклади коду
├── components/            # Vue компоненти для слайдів
├── package.json           # Залежності
└── README.md              # Інструкції
```

### 3.3 Вимоги до навколишнього середовища

- Node.js 18+
- npm або pnpm
- Сучасний браузер для перегляду презентації
- IDE з підтримкою AI-асистента для демонстрації

## 5. Незункціональні вимоги

### 5.1 Якість

- Контент має бути актуальним та точним
- Приклади коду мають бути працюючими
- Всі посилання мають бути валідними

### 5.2 Доступність

- Презентація має бути читабельною на різних екранах
- Шпаргалка має бути друкованою
- Матеріали мають бути доступні онлайн

### 5.3 Продуктивність

- Презентація має завантажуватися швидко
- Анімації мають бути плавними
- Експорт PDF має працювати стабільно

## 6. Критерії успіху

### 6.1 Метрики

- Учасники зможуть ефективно формулювати запити до AI
- Учасники зрозуміють основні правила роботи з Agentic IDE
- Учасники отримають практичний досвід під час демонстрації
- Матеріали будуть використовуватися після воркшопу

### 6.2 Оцінка

- Зворотний зв'язок від учасників
- Кількість запитань під час Q&A
- Використання матеріалів після воркшопу
