# План: Cursor Commands для Agentic IDE

## Мета

Створити набір користувацьких Cursor Commands у форматі Markdown файлів у директорії `.cursor/commands/`, які можна викликати через префікс `/` в чаті Cursor IDE для автоматизації типових завдань роботи з Agentic IDE.

## Офіційний формат Cursor Commands

Згідно з офіційною документацією Cursor:

- Команди зберігаються у `.cursor/commands/` директорії
- Кожна команда - це окремий `.md` файл
- Команди викликаються через `/` префікс у чаті
- Файли містять опис команди, чеклисти або інструкції у форматі Markdown

## Структура команди

```markdown
# Назва команди

## Опис
Короткий опис що робить команда

## Інструкції / Чеклист
- [ ] Крок 1
- [ ] Крок 2
- [ ] Крок 3

## Приклади
Приклади використання або очікуваних результатів
```

## Детальний зміст команд

### review-code.md

```markdown
# Code Review

## Опис
Виконайте детальний аналіз коду для виявлення потенційних проблем та покращень.

## Чеклист

### Функціональність
- [ ] Чи виконує код призначену функцію?
- [ ] Чи обробляються edge cases?
- [ ] Чи правильно реалізована обробка помилок?

### Якість коду
- [ ] Чи читабельний та структурований код?
- [ ] Чи описові назви змінних та функцій?
- [ ] Чи є дублюючий або зайвий код?

### Безпека
- [ ] Чи є очевидні вразливості безпеки?
- [ ] Чи валідуються та санітизуються вхідні дані?
- [ ] Чи безпечно обробляються сенситивні дані?

### Продуктивність
- [ ] Чи оптимальна продуктивність?
- [ ] Чи немає зайвих операцій?
- [ ] Чи використовується кешування де потрібно?

### Тестування
- [ ] Чи є тести для нового коду?
- [ ] Чи покривають тести основні сценарії?
- [ ] Чи проходять всі тести?
```

### write-tests.md

```markdown
# Write Tests

## Опис
Створіть тести для перевірки функціональності коду.

## Інструкції

1. **Визначення сценаріїв тестування**
   - Окресліть основні випадки використання
   - Визначте edge cases
   - Визначте негативні сценарії

2. **Реалізація тестів**
   - Напишіть unit тести для функцій
   - Напишіть integration тести для компонентів
   - Додайте тести для edge cases

3. **Запуск та перевірка**
   - Запустіть тести
   - Переконайтеся що всі тести проходять
   - Перевірте покриття коду

## Чеклист
- [ ] Unit тести для всіх функцій
- [ ] Integration тести для компонентів
- [ ] Тести для edge cases
- [ ] Тести для обробки помилок
- [ ] Покриття коду >= 80%
```

### refactor-code.md

```markdown
# Refactor Code

## Опис
Покращте існуючий код без зміни функціональності.

## Інструкції

1. **Аналіз поточного коду**
   - Визначте проблемні місця
   - Знайдіть дублювання
   - Оцініть складність

2. **Планування рефакторингу**
   - Визначте мету рефакторингу
   - Плануйте зміни
   - Переконайтеся що тести покривають код

3. **Виконання рефакторингу**
   - Винесіть дублювання
   - Спростите складну логіку
   - Покращте читабельність
   - Додайте типізацію

4. **Перевірка**
   - Запустіть тести
   - Перевірте функціональність
   - Переконайтеся що нічого не зламалося

## Чеклист
- [ ] Код став більш читабельним
- [ ] Видалено дублювання
- [ ] Складність зменшена
- [ ] Всі тести проходять
- [ ] Функціональність не змінилася
```

### fix-bug.md

```markdown
# Fix Bug

## Опис
Виправте помилку в коді.

## Інструкції

1. **Діагностика**
   - Відтворіть помилку
   - Проаналізуйте stack trace
   - Знайдіть джерело проблеми

2. **Виправлення**
   - Виправте помилку
   - Додайте обробку edge cases
   - Додайте валідацію якщо потрібно

3. **Тестування**
   - Напишіть тест що відтворює помилку
   - Переконайтеся що тест проходить після виправлення
   - Запустіть всі тести

## Чеклист
- [ ] Помилка виправлена
- [ ] Додано тест що покриває випадок
- [ ] Всі тести проходять
- [ ] Додано обробку подібних випадків
- [ ] Оновлено документацію якщо потрібно
```
