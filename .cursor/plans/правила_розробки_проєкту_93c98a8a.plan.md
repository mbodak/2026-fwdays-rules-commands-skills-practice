## Детальний зміст RULES.md

### Розділ 1: Загальні принципи

- Код має бути читабельним та зрозумілим
- Контент має бути актуальним та точним
- Документація має бути повною та зрозумілою
- Принцип DRY (Don't Repeat Yourself)
- Принцип KISS (Keep It Simple, Stupid)

### Розділ 3: Git Workflow

- **Main branch:** `main` - стабільна версія
- **Feature branches:** `feature/назва-фічі` - для нових фіч
- **Fix branches:** `fix/назва-бага` - для виправлень
- **Docs branches:** `docs/назва-зміни` - для документації
- **Commit messages:** Conventional Commits формат
- **PR process:** Обов'язковий code review перед мерджем

### Розділ 4: Coding Standards

- Markdown: використання стандартного синтаксису
- Slidev: дотримання структури frontmatter
- Vue компоненти: Composition API, TypeScript
- Іменування: camelCase для змінних, PascalCase для компонентів
- Коментарі: українською мовою для контенту, англійською для коду

### Розділ 5: Контент

- Слайди: чітка структура, логічна послідовність
- Приклади коду: працюючі, з коментарями
- Вправи: чіткі інструкції, очікувані результати
- Документація: актуальна, повна, зрозуміла

### Розділ 6: Якість

- Чеклист перед комітом
- Тестування на локальному середовищі
- Перевірка посилань
- Spell checking
- Перевірка форматування

### Розділ 7: Залежності

- Використання фіксованих версій для production
- Регулярні оновлення залежностей
- Перевірка вразливостей (npm audit)
- Документування змін у залежностях

### Розділ 8: Документація

- README: оновлювати при змінах

- Changelog: вести історію змін

### Розділ 9: Процеси

- Workflow додавання контенту
- Workflow оновлення контенту

## Конкретні правила

### Git Commit Messages

Формат: `type(scope): subject`

Типи:

- `feat`: нова функціональність
- `fix`: виправлення бага
- `docs`: зміни в документації
- `style`: форматування коду
- `refactor`: рефакторинг
- `test`: додавання тестів
- `chore`: рутинні завдання

Приклади:

```
feat(slides): додано слайд про правила роботи
fix(exercises): виправлено помилку в прикладі коду
docs(readme): оновлено інструкції встановлення
```

### Структура слайдів

```markdown
---
theme: seriph

---

# Заголовок

Контент слайду

<!-- Нотатки для презентатора -->
```

- Утиліти: `camelCase.ts`
- Константи: `UPPER_SNAKE_CASE.ts`

## Чеклисти

### Перед комітом

- [ ] Код відформатований
- [ ] Всі посилання працюють
- [ ] Презентація запускається локально
- [ ] Немає помилок лінтера
- [ ] Документація оновлена
- [ ] Commit message відповідає конвенціям

### Перед PR

- [ ] Всі зміни протестовані
- [ ] README оновлений (якщо потрібно)
- [ ] PR опис містить деталі змін
- [ ] Відзначено reviewers

### Перед релізом

- [ ] Всі завдання виконані
- [ ] Тести пройдені
- [ ] Документація актуальна
- [ ] Версія оновлена
- [ ] Changelog оновлений
- [ ] Презентація протестована на різних пристроях
