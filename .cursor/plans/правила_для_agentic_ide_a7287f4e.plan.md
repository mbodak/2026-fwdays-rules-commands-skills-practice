# План: Правила для роботи з Agentic IDE

## Мета

Створити комплексний набір правил для ефективної роботи з Agentic IDE (AI-асистентами в IDE) у форматі Markdown файлів, які будуть використовуватися як основний контент воркшопу та навчальні матеріали.

## Структура правил

### 1. Основні правила (docs/rules/basic-rules.md)

- Чіткість інструкцій
- Ітеративний підхід
- Контроль якості
- Контекст та специфічність

### 2. Правила формулювання запитів (docs/rules/query-rules.md)

- Структура запитів
- Використання контексту
- Вказівка технологій та бібліотек
- Приклади правильних та неправильних запитів

### 3. Правила роботи з кодом (docs/rules/code-rules.md)

- Створення нового коду
- Рефакторинг існуючого коду
- Виправлення помилок
- Оптимізація та покращення

### 4. Правила безпеки та якості (docs/rules/quality-rules.md)

- Перевірка згенерованого коду
- Тестування
- Безпека та вразливості
- Дотримання стандартів проекту

### 5. Правила для різних типів задач (docs/rules/task-specific-rules.md)

- Створення компонентів
- Робота з API
- Робота з базами даних
- Тестування
- Документація

### 6. Best Practices (docs/rules/best-practices.md)

- Патерни ефективних запитів
- Типові помилки та як їх уникати
- Оптимізація робочого процесу
- Покращення результатів

## Детальний зміст кожного файлу

### basic-rules.md

#### 1. Чіткість інструкцій

- Конкретність замість загальності
- Вказівка очікуваного результату
- Використання прикладів
- Уникання двозначностей

#### 2. Ітеративний підхід

- Розбиття складних задач
- Поетапна перевірка
- Уточнення та корегування
- Навчання на помилках

#### 3. Контроль якості

- Обов'язкова перевірка коду
- Тестування функціональності
- Дотримання стандартів
- Code review процес

#### 4. Контекст та специфічність

- Надання достатнього контексту
- Вказівка файлів та функцій
- Згадування технологій
- Пояснення бізнес-логіки

### query-rules.md

#### 1. Структура запитів

- Формат: Дія + Об'єкт + Контекст + Очікуваний результат
- Використання конкретних термінів
- Вказівка технічних деталей
- Приклади структури

#### 2. Використання контексту

- Вказівка поточного стану коду
- Згадування пов'язаних файлів
- Опис проблеми або мети
- Історичний контекст (якщо потрібно)

#### 3. Технології та бібліотеки

- Явна вказівка фреймворків
- Версії бібліотек (якщо важливо)
- Стилі кодування
- Існуючі паттерни проекту

#### 4. Приклади

- Правильні запити з поясненнями
- Неправильні запити та чому вони погано
- Порівняння "до" та "після"

### code-rules.md

#### 1. Створення нового коду

- Вказівка мови програмування
- Вказівка фреймворку/бібліотеки
- Опис функціональності
- Вимоги до структури

#### 2. Рефакторинг

- Вказівка файлу та рядків
- Опис мети рефакторингу
- Збереження функціональності
- Покращення читабельності

#### 3. Виправлення помилок

- Опис помилки
- Вказівка місця помилки
- Очікувана поведінка
- Контекст помилки

#### 4. Оптимізація

- Вказівка метрики для оптимізації
- Поточні проблеми
- Обмеження та вимоги
- Очікуваний результат

### quality-rules.md

#### 1. Перевірка коду

- Читання згенерованого коду
- Розуміння логіки
- Перевірка на помилки
- Порівняння з очікуваннями

#### 2. Тестування

- Написання тестів
- Перевірка edge cases
- Інтеграційне тестування
- Перевірка продуктивності

#### 3. Безпека

- Перевірка вразливостей
- Валідація вхідних даних
- Обробка помилок
- Захист сенситивних даних

#### 4. Стандарти проекту

- Дотримання coding style
- Використання існуючих паттернів
- Консистентність з кодбазою
- Документація

### task-specific-rules.md

#### 1. Створення компонентів

- Вказівка типу компонента
- Пропси та інтерфейси
- Стилізація
- Поведінка та взаємодія

#### 2. Робота з API

- Типи запитів
- Обробка відповідей
- Обробка помилок
- Типізація даних

#### 3. Робота з базами даних

- Структура даних
- Запити та міграції
- Оптимізація
- Безпека

#### 4. Тестування

- Типи тестів
- Покриття коду
- Моки та стаби
- Інтеграційні тести

#### 5. Документація

- JSDoc коментарі
- README файли
- API документація
- Приклади використання

### best-practices.md

#### 1. Патерни запитів

- Шаблони для типових задач
- Ефективні формулювання
- Комбінування запитів
- Послідовність дій

#### 2. Типові помилки

- Занадто загальні запити
- Відсутність контексту
- Ігнорування результатів
- Відсутність перевірки

#### 3. Оптимізація процесу

- Створення шаблонів
- Збереження успішних запитів
- Навчання на досвіді
- Автоматизація рутини

#### 4. Покращення результатів

- Ітеративне уточнення
- Використання прикладів
- Надання зворотного зв'язку
- Адаптація під проект

## Формат файлів

Всі файли правил мають слідувати такій структурі:

````markdown
# Назва розділу правил

## Короткий опис
Короткий опис що покривають ці правила

## Правило 1: Назва
Опис правила
- Пункт 1
- Пункт 2

### Приклади
```code
Приклад коду або запиту
````

### ❌ Неправильно

Приклад неправильного підходу

### ✅ Правильно

Приклад правильного підходу

## Правило 2: Назва

...

```
