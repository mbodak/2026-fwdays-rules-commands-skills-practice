# План: Команди для роботи з Agentic IDE

## Мета

Створити комплексний довідник найважливіших команд для роботи з Agentic IDE, включаючи шаблони запитів, практичні приклади та рекомендації щодо використання.

## Категорії команд

### 1. Створення коду (Creation Commands)

Найважливіші команди для створення нового коду

#### 1.1 Створення компонентів

- React/Vue компоненти
- Функції та класи
- Модулі та утиліти
- API endpoints

#### 1.2 Створення структур даних

- Типи та інтерфейси
- Моделі даних
- Константи та конфігурації

#### 1.3 Створення тестів

- Unit тести
- Integration тести
- E2E тести

### 2. Рефакторинг (Refactoring Commands)

Команди для покращення існуючого коду

#### 2.1 Структурний рефакторинг

- Винесення логіки
- Розбиття на модулі
- Об'єднання дублікатів

#### 2.2 Покращення якості

- Оптимізація продуктивності
- Покращення читабельності
- Додавання типізації

#### 2.3 Модернізація

- Оновлення синтаксису
- Міграція на нові API
- Заміна застарілих паттернів

### 3. Пошук та аналіз (Search & Analysis Commands)

Команди для дослідження кодової бази

#### 3.1 Пошук

- Знаходження використання
- Пошук залежностей
- Пошук паттернів

#### 3.2 Аналіз

- Аналіз складності
- Виявлення проблем
- Аналіз продуктивності

#### 3.3 Документація

- Генерація документації
- Пояснення коду
- Створення діаграм

### 4. Виправлення помилок (Debugging Commands)

Команди для діагностики та виправлення

#### 4.1 Діагностика

- Аналіз помилок
- Трасування проблем
- Логування

#### 4.2 Виправлення

- Виправлення багів
- Обробка помилок
- Edge cases

#### 4.3 Покращення

- Додавання валідації
- Покращення обробки помилок
- Додавання fallback

### 5. Оптимізація (Optimization Commands)

Команди для покращення продуктивності

#### 5.1 Продуктивність

- Оптимізація запитів
- Кешування
- Lazy loading

#### 5.2 Розмір

- Мінімізація bundle
- Tree shaking
- Code splitting

#### 5.3 Пам'ять

- Виправлення витоків пам'яті
- Оптимізація структур даних
- Garbage collection

### 6. Тестування (Testing Commands)

Команди для роботи з тестами

#### 6.1 Створення тестів

- Unit тести
- Integration тести
- Mock та стаби

#### 6.2 Покриття

- Аналіз покриття
- Додавання тестів для непокритого коду
- Критичні шляхи

#### 6.3 Покращення

- Рефакторинг тестів
- Оптимізація тестів
- Покращення читабельності

### 7. Документація (Documentation Commands)

Команди для створення документації

#### 7.1 Код документація

- JSDoc коментарі
- Type definitions
- API документація

#### 7.2 README та гайди

- Оновлення README
- Створення гайдів
- Приклади використання

#### 7.3 Діаграми

- Створення діаграм
- Візуалізація архітектури
- Flowcharts

3. "Створи API endpoint [шлях] для [мета]"
4. "Напиши тест для [компонент/функція]"

### Категорія: Рефакторинг

5. "Рефактори [файл/функція] для [мета]"

7. "Оптимізуй [функція/компонент] для [метрика]"
8. "Перепиши [код] використовуючи [паттерн/бібліотека]"

### Категорія: Виправлення

14. "Додай обробку помилок для [функція]"
15. "Виправ [тип помилки] в [місце]"
16. "Додай валідацію для [вхідні дані]"

### Категорія: Оптимізація

17. "Оптимізуй [функція] для покращення [метрика]"
18. "Зменши розмір bundle для [модуль]"
19. "Покращ продуктивність [компонент]"

## Формат команди

Кожна команда має містити:

```markdown
## Назва команди

### Опис
Короткий опис що робить команда

### Шаблон
```

[Шаблон команди з плейсхолдерами]

````

### Параметри
- `параметр1` - опис
- `параметр2` - опис

### Приклади

#### Приклад 1: [Контекст]
```markdown
[Точний приклад запиту]
````

**Результат:** [Що очікується]

#### Приклад 2: [Контекст]

```markdown
[Точний приклад запиту]
```

**Результат:** [Що очікується]

### ❌ Неправильно

```markdown
[Приклад неправильного запиту]
```

**Чому погано:** [Пояснення]

### ✅ Правильно

```markdown
[Приклад правильного запиту]
```

**Чому добре:** [Пояснення]

### Поради

- Порада 1
- Порада 2

```
